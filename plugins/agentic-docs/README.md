# Agentic-Docs Plugin

Complete lifecycle management for AI-friendly documentation in OpenShift/Kubernetes repositories.

## Overview

This plugin provides comprehensive agentic documentation capabilities for OpenShift repositories, following the Agent Knowledge Framework defined in [agentic-docs-guide](https://github.com/Prashanth684/agentic-docs-guide/blob/main/AGENTIC_DOCS_FRAMEWORK.md).

## Commands

All commands use the `/agentic-docs:` namespace:

### `/agentic-docs:create`

Generate complete agentic documentation for a repository.

**Usage:**
```bash
/agentic-docs:create [<repo-path>]
```

**What it does:**
- Analyzes repository structure
- Generates documentation following the framework structure
- Creates all required files (DESIGN.md, DEVELOPMENT.md, TESTING.md, etc.)
- Each file is generated by a dedicated skill/prompt
- Logs all data sources, tools, and skills used

**Input:** Repository files only (no GitHub/JIRA ingestion)

**Output:**
- Complete agentic documentation in `agentic/` directory
- CLI logging with real-time progress
- Timestamped log file in `logs/`

### `/agentic-docs:create-knowledge-graph`

Generate knowledge graph from existing agentic documentation.

**Usage:**
```bash
/agentic-docs:create-knowledge-graph [<repo-path>]
```

**What it does:**
- Reads existing agentic documentation
- Generates NetworkX knowledge graph
- Creates graph.json in `agentic/knowledge-graph/`

**Requirements:** Must be run AFTER `/agentic-docs:create`

### `/agentic-docs:validate`

Validate existing agentic documentation structure.

**Usage:**
```bash
/agentic-docs:validate [<repo-path>]
```

**What it does:**
- Validates directory structure (required directories exist)
- Verifies file presence (required files present)
- Checks file naming conventions

**Configuration:** All validation parameters are defined in `config/validation.yaml`

**Output:**
- Validation summary report
- Repository name and timestamp
- Pass/fail status for structure checks

### `/agentic-docs:evaluate`

Test documentation quality with coding scenarios.

**Usage:**
```bash
/agentic-docs:evaluate [<repo-path>]
```

**What it does:**
- Spawns coding sub-agents with benchmark tasks
- Tests documentation usability
- Evaluates execution plans
- Displays agent execution flow

**Output:**
- Primary agent and sub-agents display
- Transparent execution flow logs
- Data flow between components
- Timestamped evaluation logs

## Architecture

### Content Generation

The agentic-docs plugin uses a **skill-driven** architecture with **bash-based utilities**:

**Structure Creation:**
- `skills/create/scripts/structure_generator.sh` creates the directory structure and placeholder files
- Pure bash implementation (no Python dependencies)

**Content Generation:**
- Individual skills generate each documentation file
- Skills analyze the repository and generate tailored content
- Skills handle all content generation (no separate generator scripts)

**Validation:**
- `skills/validate/scripts/validate.sh` validates structure against configuration
- Pure bash implementation with YAML config parsing

**Example:**
- `DESIGN.md` → Generated by `/generate-design-md` skill
- `DEVELOPMENT.md` → Generated by `/generate-development-md` skill
- `TESTING.md` → Generated by `/generate-testing-md` skill
- And so on...

The `/agentic-docs:create` skill orchestrates all generation skills in the correct order.

### Validation Configuration

All validation parameters are user-configurable via `config/validation.yaml`. The agent **cannot** modify these parameters during execution (strict immutability enforced).

### Logging

Every command execution produces:
- **CLI output:** Real-time progress and results
- **Timestamped log file:** Complete execution trace
- **Execution metrics:** Performance and usage analytics

### Telemetry

For telemetry and metrics, use:
- **`/metrics:ai-docs-telemetry`** - Primary telemetry command
- **Claude Code JSONL logs** - Session logs at `~/.claude/projects/<session-id>.jsonl`
- **Promptfoo** - Evaluation framework (configured in `promptfooconfig.yaml`)

## Quality Guidelines

### Structure Requirements
- Complete directory structure (`agentic/` with all required subdirectories)
- All required files present (AGENTS.md, ARCHITECTURE.md, core docs, etc.)
- Proper file naming conventions

### Validation
Use `/agentic-docs:validate` to check that the documentation structure meets requirements defined in `config/validation.yaml`.

## Configuration

Validation parameters are defined in `config/validation.yaml`.

**The validation agent CANNOT modify this file** - you must edit it directly.

### Quick Configuration

Edit the configuration file:

```bash
vim plugins/agentic-docs/config/validation.yaml
```

Parameters:
- **Required directories**: List of directories that must exist
- **Required files**: List of files that must be present

### Example Configuration

```yaml
# config/validation.yaml
structure:
  required_directories:
    - agentic/
    - agentic/design-docs/
    - agentic/domain/

  required_files:
    - AGENTS.md
    - ARCHITECTURE.md
    - agentic/DESIGN.md
```

## Development

### Runtime Dependencies

**Core Scripts**: No Python dependencies required - all core functionality uses bash scripts.

**Testing Only**: Python dependencies for running tests:

```bash
# Install test dependencies
pip install -r tests/requirements.txt
```

**Test packages:**
- `PyYAML>=6.0` - YAML parsing in legacy tests
- `pytest>=7.0.0` - Testing framework

### Testing

```bash
# Test create command
/agentic-docs:create /path/to/test-repo

# Test validation
/agentic-docs:validate /path/to/test-repo

# Test evaluation  
/agentic-docs:evaluate /path/to/test-repo

# Test knowledge graph
/agentic-docs:create-knowledge-graph /path/to/test-repo

# Run Python unit tests
cd plugins/agentic-docs
pytest tests/
```
